typedef struct lise_event_context lise_event_context

The context of an event.

Stores the "parameters" for an event. The size of an event context is 128 bytes. These 128 bytes are split up in different ways to allow for easier access to parameter.

It is possible to store the following:

  • 2 signed 64-bit integers
  • 2 unsigned 64-bit integers
  • 2 doubles
  • 4 signed 32-bit integers
  • 4 unsigned 32-bit integers
  • 4 floats
  • 8 signed 16-bit integers
  • 8 unsigned 16-bit integers
  • 16 signed 8-bit integers
  • 16 unsigned 8-bit integers
  • 16 characters

◆ lise_event_codes

enum lise_event_codes

This enum contains all the types of engine specific events.

Enumerator
LISE_EVENT_ON_WINDOW_CLOSE 

Called when the game window should close.

Provides an empty event context.

LISE_EVENT_ON_MOUSE_BUTTON_DOWN 

Called when the left mouse button is pushed down.

Provides an empty event context.

LISE_EVENT_ON_MOUSE_BUTTON_UP 

Called when the left mouse button is released.

Provides an empty event context.

LISE_EVENT_ON_MOUSE_MOVE 

Called when the mouse is moved.

i32[0] will contain the x-cooordinate of the new mouse position.
i32[1] will contain the y-cooordinate of the new mouse position.

LISE_EVENT_ON_MOUSE_WHEEL_MOVE 

Called when the mouse wheel is moved.

Mouse wheel movement is normalized.

i8[0] will contain the direction of movement. -1 if the mouse wheel moved down and 1 if it moved up.

LISE_EVENT_ON_KEY_DOWN 

Called when a key is pressed.

u32[0] will contain the key that was pressed. See lise_keys for all available keys.

LISE_EVENT_ON_KEY_UP 

Called when a key is released.

u32[0] will contain the key that was released. See lise_keys for all available keys.

LISE_EVENT_ON_WINDOW_RESIZE 

Called when the window is resized.

u32[0] will contain the new width of the window.
u32[1] will contain the new height of the window.

LAPI void lise_event_register ( uint16_t  event_code)

Registers an event code.

Parameters
event_codeThe event code to register.
Note
The event codes 0 up to and including 1024 are preserved for engine event codes. It is still possible to register an event within this range because not every code within the range is in use yet, but it is not recommended to do so because it could create conflicts with future updates if a new engine event has been registered.
